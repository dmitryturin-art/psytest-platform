# Дизайн: Исправление расчётов СМИЛ, восстановление классического графика и UI-полировка

**Дата:** 2026-06-22  
**Статус:** Утверждён пользователем  
**Ветка:** `fix/smil-scoring-and-classic-chart`  
**Приоритет:** Критический (функциональность) + Средний (UI)

---

## 1. Контекст и цели

**Проект:** PsyTest Platform — профессиональный инструмент психологического тестирования для клинических психологов.

**Критические проблемы:**
1. **Ключи разметки СМИЛ неверны** → сырые баллы считаются неправильно → T-баллы выходят за диапазон (120 вместо 70)
2. **График профиля заменён на Chart.js** вместо классического MMPI с разрывом
3. **Дополнительные шкалы не отображаются** (данные есть, шаблон сломан)
4. **BDI и HADS не зарегистрированы** в БД (модули есть, в списке тестов нет)
5. **UI выглядит непрофессионально** для клинического использования

**Цель:** Восстановить корректность расчётов СМИЛ, классический график по эталону psytests.org, довести до профессионального уровня UI.

---

## 2. Архитектурные решения

### 2.1 Последовательность работ (по приоритету)

```
1. График (быстро, визуальная обратная связь)
   ↓
2. Ключи СМИЛ (критично для корректности)
   ↓
3. Калькуляторы + тесты (валидация на эталонах)
   ↓
4. Дополнительные шкалы (углубленный анализ)
   ↓
5. Регистрация BDI/HADS (расширение тестов)
   ↓
6. UI-полировка (профессиональный вид)
```

**Обоснование:** Сначала данные (чтобы график показывал правду), потом визуализация, потом юзабилити.

### 2.2 Принципы дизайна

**Профессиональный клинический инструмент:**
- График MMPI **строго по эталону** (как на psytests.org, как в клинической практике)
- Никаких "современных улучшений" графика (tooltip, анимации) — **только точность**
- UI чистый, без иконок и украшений — для работы, не для баловства
- Читаемость и удобство для психолога важнее "красоты"

### Задача 4: Исправление дополнительных шкал

#### Проблема
- Данные есть в `calculated_results.additional_scores` (23 шкалы)
- Секция "Дополнительные шкалы" показывает только заголовок таблицы
- Данные не передаются в шаблон

#### Решение

**Шаг 1:** Проверить `SmilModule::buildAdditionalScalesData()`
```php
private function buildAdditionalScalesData(array $additionalScores): array
{
    $categories = [];
    // Группировка по категориям (Welsh, Content, Clinical)
    foreach ($categoryNames as $category => $name) {
        $items = [];
        foreach ($additionalScores as $code => $score) {
            if (in_category($code, $category)) {
                $items[] = [
                    'code' => $code,
                    'name' => $score['name'],
                    'raw' => $score['raw'],
                    't_score' => $score['t'],
                    'level' => $this->getScoreLevel($score['t']),
                ];
            }
        }
        $categories[] = ['name' => $name, 'items' => $items];
    }
    return ['categories' => $categories];
}
```

**Шаг 2:** Проверить шаблон `blocks/scales-table.twig`
- Должен поддерживать группировку по категориям
- Если данные передаются как `categories`, а шаблон ожидает `scales` → несовпадение
- Исправить или адаптировать шаблон

**Шаг 3:** Категории дополнительных шкал
```

Dominance (2): Do, Re
Content (15): ANX, FRS, OBS, DEP, HEA, BIZ, ANG, CYN, ASP, TPA, LSE, SOD, FAM, WRK, TRT
Clinical (4): Es, MAC, O-H, Pk
```

**Шаг 4:** Валидация

- Таблица с колонками: Код | Название | Сырой балл | T-балл | Уровень

### Задача 6: UI-полировка (профессиональный стиль)

#### Принципы
- **Не модный, а профессиональный** — как в научных публикациях, медицинских системах
- **График MMPI строго по эталону** — без tooltip, анимаций, интерактивности
- **Без иконок и украшений** — только чёткая информация
- **Удобство для психолога** — читаемость, навигация, печать

#### 6.1 Главная страница (`/tests`)

**Текущее состояние:** Карточки тестов (test-card)

**Улучшения:**
- Чистый табличный или списочный вид (не карточки с тенями)
- Строгая типографика: Georgia / PT Serif для заголовков, PT Sans для текста
- Информация: название, описание (2 строки), кол-во вопросов, время
- Кнопка "Пройти тест" без излишнего оформления
- Цветовая схема: белый фон, тёмный текст (#333), акценты синие (#2c5aa0)

**Антипаттерны (не делать):**
- ❌ Иконки тестов
- ❌ Hover-эффекты с тенями
- ❌ Градиенты и анимации
- ❌ "Модный" card-дизайн

#### 6.2 Страница прохождения теста (`/test/{slug}`)

**Текущие проблемы:**
- Radio buttons слишком мелкие и непонятные
- Нет прогресса
- Непонятно сколько осталось

**Улучшения:**
